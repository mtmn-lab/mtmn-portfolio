# 潜入DXログ

## Purpose

現場業務を理解しながら、
「どこに改善余地があるか」
「どの技術で負荷軽減できるか」
を観察・整理する目的で記録。

業務効率化だけでなく、
情報整理・ナレッジ共有・データ活用まで含めて
改善視点を残している。

詳細な改善ログ・設計メモは個人管理環境で継続記録中

---

# Day1

## Summary
- 業務説明・環境設定・夕方ルーティン業務を実施
- 使用システムやデータ保存場所が複数に分散している状況を確認
- URL・ログイン情報・業務情報がExcel中心で管理されていることを把握
- Microsoft 365 / Power Platform利用可能性を調査
- SharePoint・Power Automate・Power BI活用余地を確認
- 情報整理・ナレッジ共有・定型作業自動化に改善余地を感じた

---

# Day2

## Summary
- 商品実績確認業務および各種設定変更作業を実施
- Excel転記・複数サイト更新・多段ログインなど手作業負荷が高いことを確認
- 前職でのDX経験を共有し、SharePoint活用による情報整理案を提案
- Power BI / Power Queryによる自動集計・可視化余地を確認
- 商品コードマスタ整備やデータ取得方式整理の重要性を認識
- ECサイト横断分析や実績データ活用による改善提案可能性を感じた
- 単純効率化だけでなく、分析基盤整備による付加価値創出余地を確認

---

# Day3

## Summary
- Power BIによるKPI可視化サンプルを作成
- Power BI導入イメージを軽く提案
- Power Automate利用可否を確認
- 現場の認証管理・情報管理方法を確認
- 手作業・転記作業・ログイン作業の負荷が非常に高いことを実感
- 小規模自動化だけでも改善余地が大きいと判断

---

# Day4

## Summary

- ルーティン業務・画像更新作業を実施
- Power Automate Desktopで自動ログインフローを試作
- Power BI Service環境および既存分析機能を調査
- 改善方針を「分析追加」から「現場負荷軽減」中心へ再整理
- 小規模自動化でも高い改善余地を確認

## Tech Memo

- PADによる多段ログイン自動化検証
- 認証情報管理方式・Power BI Service制約を確認
- 定型業務自動化およびPower Platform活用方針を整理
