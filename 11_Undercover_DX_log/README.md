# 潜入DXログ

実際の現場で感じた「業務改善ポイント」や「DX視点」を記録したログ。
Power Platform / BI / 自動化の観点から、現場改善アイデアを整理していきます。

# Day1

## 実施内容
- 業務説明
- 使用ツール / システム紹介
- NAS設定
- 業務フローの一部説明
- 夕方ルーティン業務の実践

## 現場で気づいたこと
- 使用システムや管理方法が複数存在している
- NASやローカル、各種管理画面など、データ保存場所が分散している
- URL / ログイン情報 / 業務情報などがExcel中心で管理されている
- 新人教育は口頭説明が中心で、情報整理やナレッジ共有基盤は未整備

## 技術観点で確認したこと
- Microsoft 365環境確認
- Power Platform利用可否確認
- Power BI / Power Automate利用可能性確認
- SharePoint運用
- ローカル環境 / 保存構成確認

## 改善アイデア
- SharePoint Listによる情報整理
- SharePoint チームサイトの確認
- ログイン情報管理改善
- 新人教育内容のナレッジ化
- 定型ログイン作業の自動化
- ルーティン業務のPower Automate化

## 所感
業務理解より先に、
「なぜこの運用が手動なのか？」
という改善ポイントの観察が始まっていた。

現場運用は人力前提で回っている部分が多く、
自動化・標準化余地が大きいと感じた。


# Day2

## 実施内容

### 商品実績確認業務
- 商品マスタから対象データ抽出
- 管理システム上で商品コード検索
- 実績数値をExcelへ転記
- 増加傾向商品の確認作業を実施

### 各種設定変更業務
- 複数管理サイトへログイン
- 商品受付関連の設定変更作業を実施

### 業務改善に関する会話
- 前職でのDX / 業務改善経験を共有
- SharePoint活用による情報整理案を提案
    - 新人教育資料整理
    - URL / パスワード管理
    - 情報共有基盤整備

---

## 現場で気づいたこと
- 同一データを複数サイトへ手入力している
- ログイン回数が非常に多い
- Excel転記作業が多く、確認作業も属人的
- 商品コード管理が整理されていない
- 業務ルールや管理方法が個人依存になっている

---

## 分析・改善アイデア

### 実績確認業務
- Power Query / Power BIによる自動集計化
- 前日比・増減率の自動可視化
- 商品コードマスタ整備
- データ取得方式の調査

### 設定変更業務
- Power Automate Desktopによる半自動化
- 管理データを基準にした一括更新構成検討
- 更新対象一覧の自動生成

### 情報整理
- SharePoint Team Site活用
- 教育資料のナレッジ化
- URL / ID管理改善
- 業務フロー可視化

---

## データ活用構想
- ECサイト横断の商品実績分析
- 商品別人気ランキング
- 事業者別実績分析
- 動きの少ない商品の抽出
- 季節要因・トレンド分析

また、単なる集計ではなく、

- 出品改善提案
- 商品見直し
- 露出改善
- 在庫調整
- 新規提案

など、実績データを起点とした改善提案へ発展可能と感じた。

さらに、分析基盤を整備することで、

- 商品登録支援
- 事業者支援
- 自治体支援
- 運営効率化

など、付加価値提案にも繋げられる可能性があると考えた。

---

## 技術観点メモ
- Power BIで十分に可視化可能な業務と判断
- マスタ整備が重要
- 元データ取得方法次第で自動化難易度が大きく変わる
- API利用可否確認が今後の鍵

---

## 所感
現場では「毎日やること」がそのまま運用として固定化されている印象。

特に、

- 複数サイトへの同一内容更新
- Excelへの転記作業
- 手動ログインの繰り返し

など、自動化・標準化余地が非常に大きいと感じた。

また、単なる業務効率化だけではなく、
データ活用による分析・改善提案まで繋げられる可能性を感じた。
